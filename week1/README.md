# 클린 코드 1장 깨끗한 코드
1. 코드가 존재하리라
기술이 발전해도 결국 AI 프롬프트를 잘 넣어야 하는 것처럼 우리의 요구사항을 말하는 것이 필요하므로 코드는 항상 존재하리라.
2. 나쁜 코드
나쁜 코드로 회사가 망한다. 나중은 결코 오지 않는다.
3. 나쁜 코드로 치르는 대가
나쁜 코드가 쌓일 수록 개발 속도가 느려진다.
재설계를 해도 나쁜 코드가 쌓여 또 다시 재설계를 해야 하는 상황이 올 수 있으므로 항상 깨끗한 코드를 짜야 한다.
항상 좋은 코드를 짜려는 **태도**를 가져야 한다. 상사가 빠른 일정을 요구하더라도.
기한을 맞추려면 좋은 코드를 짜야 한다. 기한을 맞추는 유일한 방법은, 그러니까 빨리 가는 유일한 방법은, 언제나 코드를 최대한 깨끗하게 유지하는 습관이다.
깨끗한 코드를 작성하는 것은 그림을 잘 그리는 것과 같다.
깨끗한 코드란?
비야네 스트롭스트룹: 우아하고 효율적인 코드.
그래디 부치: 가독성
큰 데이브 토마스: 다른 사람이 고치기 쉬운 코드.
마이클 페더스: 코드를 주의 깊게 짜라.
론 제프리스: 중복 피하라. 한 기능만 수행하라. 작게 추상화하라.
워드 커닝햄: 짐작했던 기능을 그대로 수행해야 함. 그리고 코드가 그 문제를 풀기 위한 언어처럼 보이면 아름다운 코드임.
4. 밥 아저씨 생각
밥 아저씨는 자신의 생각이 모두 옳은 것은 아니지만 분명 이익을 줄 것이라고 말한다.
5. 우리는 저자다
읽기 쉽게 짜야 한다.
6. 보이스카우트 규칙
"캠프장은 처음 왔을 떄보다 더 깨끗하게 해놓고 떠나라."
시간이 지날수록 코드가 좋아지게 만들자.
7. 프리퀄과 원칙
 이 책 읽고 PPP를 읽어보라.
8. 결론
이 책을 읽는다고 뛰어난 프로그래머가 된다는 보장은 없다. 연습만이 살길이다.
